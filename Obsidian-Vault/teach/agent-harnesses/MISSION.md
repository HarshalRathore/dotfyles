# Mission: Custom Agentic Harnesses for Looped Engineering

## Why
Build your own looped coding agent harness — not Pi, not Opencode, not a CLI wrapper, but a purpose-built system that runs agents in iterative engineering loops. The goal is to understand the architecture, design patterns, and tradeoffs of custom harnesses so you can create one that ships real features autonomously.

## Success looks like
- Explain the four requirements every engineered loop must have (goal, context, evaluation, agent)
- Compare monolithic vs multi-agent harness philosophies and know when each fits
- Design a validation contract system that prevents "tests shaped by code"
- Understand how serial vs parallel execution affects correctness in multi-day runs
- Map the roles (orchestrator, workers, validators) and their model requirements

## Constraints
- Focus on custom harnesses, not existing tools (Pi, Opencode, Claude Code CLI)
- Engineering-loop oriented — not general agentic frameworks
- Learning through lessons + HTML artifacts

## Out of scope
- Setting up or configuring Pi/Opencode/Claude Code
- LLM model selection or training
- Infrastructure/deployment of agent systems
- Security or compliance considerations

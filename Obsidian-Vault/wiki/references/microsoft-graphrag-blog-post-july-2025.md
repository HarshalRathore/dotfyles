# Microsoft GraphRAG Blog Post — LazyGraphRAG & Benchmark QED

## Summary

Jonathan Larson, lead of the GraphRAG team at [[entities/microsoft|Microsoft Research]], announced at AI Engineer World's Fair 2025 that a blog post about [[concepts/lazy-graphrag|LazyGraphRAG]] and [[concepts/benchmark-qed|Benchmark QED]] would be published the day after the talk. Both were announced as going open source. ^[extracted]

## Key Announcements

1. **LazyGraphRAG** — An evolution of GraphRAG with improved benchmark performance. The blog post was scheduled for the day after the AIEF2025 talk (July 2, 2025). Larson did not disclose the specific mechanics in his talk, focusing instead on benchmark results. ^[extracted]

2. **Benchmark QED** — A new evaluation benchmark that went open source alongside the announcement. It measures retrieval quality in GraphRAG-based systems. ^[extracted]

## Relationship to GraphRAG for Code

These announcements extend the GraphRAG for Code work demonstrated in the talk, which showed GraphRAG's superiority over standard RAG for repository-level code understanding, code translation, and feature development on large codebases. ^[inferred]

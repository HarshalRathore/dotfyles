# Dispatch from the Future: building an AI-native Company

**Speaker:** Dan Shipper
**Event:** AI Engineer World's Fair 2025 (AIEF2025)
**Video:** https://www.youtube.com/watch?v=MGzymaYBiss

## Summary

Dan Shipper, founder of Every, shares his firsthand experience building an AI-native company from the inside. Every runs four software products with just 15 people, growing MRR by double digits monthly, with 99% of code written by AI agents. Shipper introduces "compounding engineering" — a four-step loop (plan, delegate, assess, codify) where each feature makes the next easier to build — and describes how AI adoption has transformed their engineering culture toward parallel work, demo-driven development, and tacit knowledge codification.

## Key Points

### Every: A Living Lab for AI-Native Engineering

- Six business units, four software products
- 15-person team running all products
- 7,000+ paying subscribers, 100,000+ free subscribers
- MRR growing double digits every month for six months
- Only ~$1M raised total — capital-efficient
- 99% of code written by AI agents (Claude Code, Codex, Droid)
- Each app built by a single developer

### Products Mentioned

- **Quora** — AI email management app with summarization and question-answering
- **Monologue** — Speech-to-text app (similar to Whisper Flow)
- **Spiral** — Another production application built by one engineer

### Compounding Engineering

Shipper's core framework: in traditional engineering, each feature makes the next harder; in compounding engineering, each feature makes the next easier. The four-step loop:

1. **Plan** — Create detailed plans before delegation
2. **Delegate** — Tell the agent what to do
3. **Assess** — Evaluate the agent's output (tests, manual review, agent code review)
4. **Codify** — Capture tacit knowledge from planning/delegation/assessment into explicit prompts, sub-agents, and slash commands

### Key Observations

- **100% adoption threshold**: A 10x difference between 90% and 100% AI adoption; even 10% traditional engineering slows the whole org
- **Parallel work**: Engineers productively using four agent panes simultaneously
- **Code is cheap**: Enables prototyping risky ideas and running more experiments
- **Demo culture**: Moving from memos/decks toward building demos to show ideas
- **Level of stack shift**: Moving from Python/JavaScript up into English as the programming language
- **Tacit knowledge codification**: Making shared knowledge easier through the compounding loop

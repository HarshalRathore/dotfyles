# Component Inventory

A component inventory is a structured, indexed representation of a codebase's key elements — classes, variables, and their usage patterns — generated by an AI agent. ^[extracted]

## How It Works

Ask the agent to produce a step-by-step inventory of your codebase: "these are the classes, these are the variables, this is how they're used across the codebase." Once the agent has this indexed representation, its ability to find bugs improves dramatically. ^[extracted]

## Why It Helps

Without a component inventory, agents navigate codebases by reading files sequentially and lose logical links between related code. An inventory provides a persistent, indexed map that the agent can reference, maintaining cross-file reasoning even as context fills up. ^[inferred]

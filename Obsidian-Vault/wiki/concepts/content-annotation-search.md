# Content Annotation for Search

The practice of building a **comprehensive text representation** of a visual item (pin) by combining multiple feature sources — textual metadata, vision-language model extractions, and user engagement signals — to improve relevance prediction accuracy. ^[extracted]

## Five Feature Sources

Pinterest identified five complementary sources for content annotation:

| Source | Type | Description |
|---|---|---|
| **Title & description** | Textual | User-provided metadata about the pin |
| **VLM image captions** | Vision-language | Synthetic captions generated by a Vision Language Model directly from the image |
| **Board titles** | User action | Names of user-curated boards the pin was saved to (implicit categorization) |
| **Engagement queries** | User action | Search queries that led to highest engagement with the pin (implicit intent signal) |
| **Additional signals** | User action | Other engagement-based annotations |

Each source captures a different dimension of the pin's meaning: explicit metadata, visual content, user curation behavior, and user search intent. ^[extracted]

## Ablation Study Findings

Pinterest ran ablation studies to measure each feature source's contribution:

- **VLM captions alone** provide a very solid baseline for relevance prediction
- **Sequential addition** of each feature source produces incremental performance improvement
- **User action features** (board titles, engagement queries) proved particularly valuable as content annotations
- The combined five-source representation is significantly more robust than any single source

This demonstrates that enriching content representations with diverse annotation sources is a high-leverage strategy for relevance prediction. ^[extracted]

## Why User Action Signals Work

Board titles and engagement queries serve as **implicit annotations** that capture:
- **User curation intent** — how real users categorize and organize content
- **Search intent** — what users were looking for when they engaged with the content
- **Community signals** — crowd-sourced categorization that may differ from explicit metadata

These signals are generated organically by user behavior, requiring no additional annotation effort while providing rich semantic context. ^[inferred]

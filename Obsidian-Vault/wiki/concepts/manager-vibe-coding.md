# Manager Vibe Coding

The observation that managers have been "vibe coding" forever — assigning features to developers, testing the resulting app, and never reading the code. ^[extracted]

## The Pattern

Kitze describes the manager workflow:

1. Manager tells developer to implement a new feature
2. Developer makes changes to the code
3. Manager tests the app
4. Manager does not read the code

This is the essence of vibe coding: describing what you want, having someone else generate it, and evaluating the output without reading the implementation. ^[extracted]

## Implications

- Vibe coding is not a new paradigm — it's the management paradigm that has always existed
- The difference is that now developers are being asked to adopt the workflow they've been managing with
- Managers who vibe code effectively (evaluate output, give feedback, iterate) already have the core skill
- The gap is not technical — it's cultural. Convincing skeptical senior engineers to try it is the hard part. ^[extracted]

# MCP Adoption Timeline

MCP's adoption followed a clear progression from internal hack project to industry standard: launch skepticism → Cursor adoption → IDE integration → major model provider adoption. ^[extracted]

## Phase 1: Launch (November)

MCP was open-sourced in November. Early reception was skeptical — many asked "What's MCP?" or "What's MPC?" and questioned why a new protocol was needed when models could already call tools. ^[extracted]

Theo Chu noted: "when you build something zero to one, you think the launch moment is going to be really impactful. But it actually usually is not." ^[extracted]

## Phase 2: Builder Momentum

The first real momentum came when builders started using MCP to automate their own workflows. This hands-on experience changed perceptions — builders saw the value rather than just questioning the need. ^[extracted]

## Phase 3: Cursor Adoption

The major turning point came when Cursor adopted MCP. After Cursor, many other coding tools also adopted MCP, including VS Code and Source Graph. This is where "that next stage of momentum came in, where agency was given to builders to actually build MCPs for themselves." ^[extracted]

## Phase 4: Model Provider Adoption

The latest turning point was adoption by major model providers: Google, Microsoft, and OpenAI all adopted MCP. Theo described this as "really exciting to see this kind of become more and more the standard." ^[extracted]

## Why Standards Win

Theo emphasized that "standards become standards because they are actually useful to builders." The key to MCP's adoption was not marketing or partnerships — it was genuine utility for the people building with it. ^[extracted]

# kubeadm join

## Definition
`kubeadm join` is the command used to join worker nodes or additional control plane nodes to an existing Kubernetes cluster. It uses a token and certificate authority information generated by `kubeadm init` on the master node to establish a secure connection and register the new node with the cluster's control plane.

## Key Characteristics
- Requires a pre-generated token and CA information from the control plane's `kubeadm init` output.
- Is the second major step in a typical self-managed Kubernetes cluster setup, after initializing the control plane.
- Enables worker nodes to connect and participate in the cluster's container orchestration workload.
- Can be used to add both worker nodes and additional control plane nodes (for high availability).

## Applications
- Expanding the capacity of a Kubernetes cluster by adding new worker nodes.
- Setting up high-availability (HA) clusters by joining additional control plane nodes.
- Automating node provisioning and cluster scaling in self-managed Kubernetes environments.

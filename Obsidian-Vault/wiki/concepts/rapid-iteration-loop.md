# Rapid Iteration Loop

The **rapid iteration loop** is the cycle of ship → measure → learn → iterate executed at maximum speed. In a startup context, the loop speed determines how quickly you find product-market fit — faster loops mean more shots on goal per unit of time.

## The Bolt.new Loop

Bolt.new's growth from $0.7M to $20M ARR in 60 days was not a one-shot event — it was the result of a rapid iteration loop operating at high velocity:
- Ship an MVP (the "just metal" product)
- Measure real user behavior (30-40K active customers in month two)
- Learn from the data (the "clean ramp" showed the core value proposition was right)
- Iterate on infrastructure, features, and scaling

The "clean ramp" Simons describes — no jagged edges despite the insanity of early demand — suggests the iteration loop was keeping pace with growth.

## Loop Speed Determinants

1. **Team size.** Small teams with high context per head make decisions faster.
2. **Decision autonomy.** No permission chains = faster iteration.
3. **Shared values.** Low ego, high trust = fewer meetings, more shipping.
4. **Infrastructure readiness.** The underlying tech (seven years of StackBlitz development) meant the iteration loop had a solid foundation.

## The Loop vs. The Plan

Traditional product development follows a plan: research → design → build → test → ship. The rapid iteration loop replaces planning with shipping:
- Ship first, plan later
- Measure real data, not hypotheticals
- Learn from users, not from internal discussion
- Iterate based on what actually happened, not what you predicted

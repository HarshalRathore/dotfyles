# Spec-Driven Vibe Coding

Spec-driven vibe coding is the highest maturity level on the [[concepts/yolo-structured-spectrum-vibe-coding|YOLO → Structured → Spec-Driven spectrum]] of vibe coding. It represents the point where vibe coding scales to large codebases with enterprise-grade reliability, velocity, and consistency. ^[extracted]

## Characteristics

- **Large codebase compatibility** — Generated code fits into existing, complex codebases without breaking conventions or architecture
- **Scale and reliability** — The output is reliable enough for production systems at enterprise scale
- **Team velocity** — "Everyone feeling the Vibes" — the speed advantage of vibe coding is available to all team members, not just early adopters
- **Codebase fit** — Generated code "fits into the code base" — it follows existing patterns, architecture decisions, and engineering standards

## Enablers

Spec-driven vibe coding requires:

- **Full spec MCP** — Dynamic tool discovery, sampling, elicitations, and streamable HTTP enable rich, context-aware agent workflows that understand the full codebase
- **Custom tools** — MCP servers that encode domain knowledge, codebase conventions, and architectural patterns
- **Copilot Instructions** — Team-wide `.github/copilot-instructions.md` or equivalent that codify standards
- **Templates** — Starter templates that bootstrap projects with the right conventions from day one

## The Ultimate Goal

Kirshner positioned spec-driven vibes as where "most of you probably want to end up" — the state where vibe coding is not a novelty or a prototyping technique, but the default way the team writes code at scale. ^[extracted]

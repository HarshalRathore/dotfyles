# Viral Side Project Pattern

The **viral side project pattern** describes the phenomenon where a personal side project — built casually or as a procrastination exercise — is posted on Twitter and goes viral, unexpectedly launching a real product.

## The Excalidraw Case

[[Christopher Chedeau|christopher-chedeau]] built the first version of [[Excalidraw|excalidraw]] in a code sandbox on January 1, 2020, using [[Rover.js|rover-js]], after [[Zwibbler|zwibbler]] was unavailable. He posted about it on Twitter and it received 200+ likes on the first tweet, with many people replying and commenting with excitement. This unexpected community interest led to Excalidraw becoming a real product.

Three months later, COVID forced everyone remote, and Excalidraw unexpectedly became a widely adopted whiteboarding solution for distributed teams.

## Pattern Elements

1. **Accidental catalyst**: Something prevents the original plan (Zwibbler down, need to write a blog post)
2. **Quick prototype**: Build a minimal version in a sandbox (code sandbox + Rover.js)
3. **Social signal**: Post about it publicly (Twitter)
4. **Community validation**: Unexpected positive response (200+ likes, many replies)
5. **Serendipitous timing**: External event amplifies need (COVID-19 remote work)

## Implication for Open Source

This pattern suggests that open-source creators should feel comfortable sharing unfinished prototypes. The viral side project pattern shows that community interest can emerge from minimal demos — you don't need a polished product to generate traction.^{[inferred]}

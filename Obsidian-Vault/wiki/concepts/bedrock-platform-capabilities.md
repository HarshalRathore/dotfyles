# Bedrock Platform Capabilities

The comprehensive set of capabilities within [[entities/amazon-bedrock|Amazon Bedrock]] for building generative AI applications, extending beyond model access to include data automation, model customization, evaluation, and RAG infrastructure. ^[extracted]

## Capability Map

### Model Access
- Multiple foundational model providers (Amazon Nova, Anthropic, Meta, Mistral, AI21 Labs)
- Unified API for model invocation
- Choice of model per use case ^[extracted]

### Bedrock Data Automation
- Build custom data pipelines
- Transform data for AI consumption
- Handle data preparation that general-purpose tools cannot ^[extracted]

### Model Customizations
- Fine-tune models with company-specific data
- Adapt models to domain, brand voice, and business rules
- Long-term data investment baked into the model ^[extracted]

### Model Evaluation
- Evaluate model performance before and after deployment
- Compare model versions and configurations
- Quality assurance for model outputs ^[extracted]

### Bedrock Knowledge Bases
- Build [[concepts/agentic-rag|RAG]] applications quickly
- No need to write code from scratch
- Reduces time to market for retrieval-augmented applications ^[extracted]

### Responsible AI
- Infrastructure for responsible data handling
- Repeated emphasis as a core Bedrock feature
- Platform-level support for governance ^[extracted]

## Platform Philosophy

Bedrock is positioned not just as a model marketplace but as an end-to-end platform for the entire generative AI development lifecycle — from data preparation through model selection, customization, evaluation, and deployment. ^[extracted]

# Luke Alvoeiro

Luke Alvoeiro is a software engineer and multi-agent systems architect. He created Goose, an open-source coding agent, while at Block (Square). Goose was later donated to the AI Agentic AI Foundation. He now leads the core agent harness at Factory, building the "Missions" multi-agent architecture.

## Known for

- **Goose** — Open-source coding agent, one of the leading coding agents in the ecosystem. Started as a project at Block ~2.5 years ago, now at the AI Agentic AI Foundation.
- **Missions** — Three-role multi-agent system (orchestrator, workers, validators) at Factory. Uses validation contracts, structured handoffs, adversarial verification, and serial execution for multi-day autonomous runs.
- **Multi-agent taxonomy** — Proposed a five-strategy framework (delegation, creator-verifier, direct communication, negotiation, broadcast) for classifying multi-agent systems.
- **"Droid whispering"** — The skill of mentally modeling how different LLMs interact, where they fail, how failures compound over multi-day runs, and choosing which model sits in which role.

## Key stances

- The bottleneck in software engineering is human attention, not model intelligence
- Multi-agent systems should run features serially (not parallel) to avoid coordination overhead — error rate drops dramatically and correctness compounds
- Different models/providers should fill different roles (planning, implementation, validation) — no single model is best at all three
- Architecture must improve with each model generation, not be made obsolete by it
- Validation must be adversarial (agents that haven't seen the code) and defined before implementation (validation contracts)

# Christopher Harrison

Christopher Harrison (handle: `geek trainer`) is an Enterprise Advocate for [[entities/github-copilot|GitHub Copilot]], formerly at Microsoft. He co-presented with [[entities/jon-peck|Jon Peck]] at the AI Engineer World's Fair 2025 on collaborating with agents in the software development workflow. ^[extracted]

## Background

Harrison has a background as a tech trainer and has been doing developer relations work focused on developer experience. He is active across all platforms at the handle `geek trainer`. ^[extracted]

## At AIEF2025

Alongside [[entities/jon-peck|Jon Peck]], Harrison presented a hands-on lab on GitHub Copilot coding agents: ^[extracted]

- **Copilot workload spectrum** — Mapped the evolution from Code Completions → Chat → Edits → Agent Mode → Coding Agent, explaining when each mode is appropriate ^[extracted]
- **Context as good code 101** — Framed Copilot as an AI pair programmer that reads your code; good variable names, comments, and project structure are the best context you can give it ^[extracted]
- **Don't be passive-aggressive** — The core advice: if you know something important, tell Copilot. Don't expect it to infer unstated requirements. Be specific about intent. ^[extracted]
- **Coding agent architecture** — Demonstrated the issue-to-PR pipeline and its safety model (firewalled, ephemeral, branch-only, draft PRs, no self-review) ^[extracted]
- **Lab facilitation** — Managed the hands-on lab, handling logistics, troubleshooting attendee setups, and guiding participants through exercises ^[extracted]

## Key Quotes

> "You want to think about it as an AI pair programmer... not only is it wonderful marketing, it kind of is, but it's also a fantastic description of how best to think about GitHub Copilot."

> "Don't be passive aggressive with Copilot. That if there's an important piece of information that you need Copilot to consider, go ahead and make sure that you tell Copilot that."

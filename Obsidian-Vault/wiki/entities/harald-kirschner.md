# Harald Kirschner

Harald Kirschner is a [[entities/microsoft|Microsoft]] engineer working on [[concepts/vscode-agent-mode|VS Code]] and the [[concepts/model-context-protocol|MCP]] ecosystem. ^[extracted]

## MCP Advocacy

Kirschner is a prominent advocate for full [[concepts/model-context-protocol|MCP]] spec adoption beyond the tools primitive. At AI Engineer World's Fair 2025, he presented "Full Spec MCP: Hidden Capabilities of the MCP spec" ([https://www.youtube.com/watch?v=ExeD-8gFUMM](https://www.youtube.com/watch?v=ExeD-8gFUMM)), arguing that the ecosystem's focus on tools-only creates a reinforcing loop of limited client investment and limited server capabilities.

He announced that [[entities/microsoft|VS Code]] v1.10 shipped full spec support including sampling, dynamic discovery, and elicitations — moving beyond tools-only implementations.

## VS Code MCP Development

Kirschner leads VS Code's MCP integration, including:
- **Dev Mode** — debugger attachment for MCP servers running inside VS Code (works for Python and Node out of the box)
- **Tool Sets** — user-defined groupings of tools for specific workflows
- **Per-chat tool selection** — tool packer with keyboard accessibility
- **Dynamic discovery** — servers adding/removing tools at runtime

## Community Involvement

He participates in the [[concepts/model-context-protocol|MCP Steering Committee]], which met in person for the first time at the MCP Dev Summit (10 days before his AIEF2025 talk). He encourages the community to contribute to open RFCs and file issues to drive the ecosystem forward.

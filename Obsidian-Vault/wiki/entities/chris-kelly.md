# Chris Kelly

Chris Kelly is a software engineer at [[entities/augment-code|Augment Code]] with a background that includes early days at [[entities/github|GitHub]]. He brings a production software engineering perspective to the AI coding debate, emphasizing that generating code is not the same as building production software. ^[extracted]

## Key Positions

- **Code is artifact, not the job** — Software engineering is about making thousands of decisions about what to build; code is just the output. ^[extracted]
- **Every line of code carries burden** — Generated code must be maintained, debugged, and understood. More generation = more maintenance. ^[extracted]
- **Code review is the most important skill** — As AI generates more code, the industry needs to prioritize reading and evaluating code over writing it from scratch. ^[extracted]
- **Production mindset matters** — "Vibes don't cut it" at 2 AM when production is down. Production software requires rigorous engineering, not pattern matching. ^[extracted]

## Background

- Worked at [[entities/github|GitHub]] during the early days of the platform
- Now at [[entities/augment-code|Augment Code]], an enterprise AI coding platform
- Has over 20 years of experience "changing software safely"
- Carried a pager — has direct experience with production system failures

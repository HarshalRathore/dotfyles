## Factory

Factory is an AI agent platform company building infrastructure for **agent-native software development**. Founded by [[entities/eno-reyes|Eno Reyes]], the company partners with large organizations to help them transition from human-driven to agent-driven software development.

### The Platform

Factory's platform centers on **"droids"** — their term for AI agents that operate across the entire software lifecycle, not just code generation. Droids are designed to:

- Ingest and ground themselves in the development environment (codebase, git state, tools)
- Search codebases, check out branches, review recent changes
- Access both user-specific memories and organization-wide knowledge
- Produce plans and ask clarifying questions before executing
- Write code, run pre-commit hooks, lint, and generate pull requests that pass CI
- Research new model releases and integrate them into existing applications
- Analyze customer feedback patterns and technical constraints for planning

### Core Philosophy

Factory believes that true AI power in software development is only accessible when **an organization delegates the majority of tasks across the software lifecycle to agents**. This requires four hard problems to be solved:

1. An intuitive interface for managing and delegating tasks
2. Centralized context from across all engineering tools and data sources
3. Agents that consistently produce reliable, high-quality outputs
4. Infrastructure supporting thousands of agents working in parallel

### Documentation as Knowledge Base

Factory transcribes every interaction and meeting, combining these notes with droid access to architecture. Their process and documentation serve as a **knowledge base and map** for droids to learn and imitate the way the team thinks — transforming what is normally a burden into an asset for agent-native development.

# Geoffrey Huntley

Geoffrey Huntley (@GeoffreyHuntley) is a software engineer and creator of the [[concepts/ralph-coding-technique]] — a monolithic AI agent loop used to build the CURSED programming language.

## Known for

- **[[concepts/ralph-coding-technique]]** — Recursive `while :; do cat PROMPT.md | claude-code ; done` loop for greenfield software development. Used to build CURSED, a production-grade esoteric programming language.
- **Ralph's three states** — Underbaked, baked, or baked with unspecified latent behaviours.
- **"LLMs are mirrors of operator skill"** — The philosophy that tools are not the variable; operator skill is.
- **Groundhog** — A precursor technique: conversation-first specification writing before letting Ralph build.
- **Deliberate intentional practice** — A framework for developing operator skill with AI coding tools.

## Writing and philosophy

- Blog at [ghuntley.com](https://ghuntley.com/)
- Active on X ([@GeoffreyHuntley](https://x.com/GeoffreyHuntley)), LinkedIn, BlueSky
- Writes extensively on AI-assisted development, agent loops, and the "post-AGI" implications of current techniques ^[inferred]

## Key stances

- Multi-agent systems are premature complexity — monolithic single-process loops outperform multi-agent for current tooling
- Engineers are still needed, but one engineer with Ralph can displace a large majority of SWEs for greenfield work
- "If models and tools remain as they are now, we are in post-AGI territory" ^[inferred]
- The perfect prompt doesn't exist — prompts evolve through continual tuning based on observation of LLM behaviour

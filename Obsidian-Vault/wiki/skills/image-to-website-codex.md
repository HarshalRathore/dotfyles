# Image-to-Website with Codex

A practical workflow for turning visually designed website mockups (generated by AI image tools) into a working Next.js application using [[entities/codex|Codex]] as the coding agent. The approach replaces traditional wireframing with image-first design, then uses an agentic coding tool to reconstruct the site section by section.

## The Four-Step Workflow

### 1. Generate Website Images

Use [[entities/chatgpt|ChatGPT]] image generation (GPT-4 Image / images 2.0) to produce mockup images for each website section. Generate multiple variants per section to choose from.

- Run the prompt multiple times across different chats for variety ^[extracted]
- Use thinking mode for higher quality results ^[extracted]
- Generate one image per website section (horizontal aspect) ^[extracted]
- ChatGPT can generate up to 10 images at once ^[extracted]
- Refine iteratively — prompt and refine until you get the desired result ^[extracted]
- Download all images when satisfied ^[extracted]

**Example prompt structure:** "Based on [design skill/context], generate images for a website for [purpose]. The design should include [N] sections, one image per section. The website should feel polished and clean and minimalist with striking image effects and creative layouts. Use [color] as the main color. All images should be horizontal." ^[extracted]

### 2. Extract Assets from Images

Extract the individual visual elements (icons, illustrations, decorative images) embedded within each generated mockup image. This involves prompting an image model to regenerate each distinct image found inside the mockup.

**Example prompts:**
- "Extract the images on the right, generate them — total should be [N] extracted generated images, please don't change them, they should look exactly the same." ^[extracted]
- "In this image there are [N] images used. Extract them and generate all [N]. So generate [N] different images in total." ^[extracted]

### 3. Remove Backgrounds

AI-generated images typically include backgrounds. Use a background removal tool (e.g., Adobe Express Remove Background, which was free at time of writing) to isolate assets.

Note: ChatGPT no longer supports transparent image generation as a native feature, making this step necessary. ^[extracted]

### 4. Build with Codex Section by Section

Use [[entities/codex|Codex]] to reconstruct the website. Work on **one section at a time** rather than attempting the whole site in a single prompt.

**Initial section prompt template:** "Let's build a website. I will provide the images and you just clone the rest of the website. So do NOT generate or build the asset in this image. Focus on the components and details. Here is the hero section. Copy it. After that give me the dir of the folder where I need to put my images in. Use React + Next.js. Start local dev server afterwards." ^[extracted]

**Iterative refinement:** When encountering alignment or overlapping issues, screenshot the result and provide it to Codex for fixes. ^[extracted]

**Patience is required** — a single prompt rarely produces a polished result. The author's advice: "Good things come to those who wait." ^[extracted]

## Next Steps (Post-Initial Build)

After the basic structure is working, refine with:

- **Scroll animations:** "Add animations when scrolling into a new section. Should feel smooth and clean." ^[extracted]
- **Hover animations** ^[extracted]
- **Detail and responsiveness** — fix sizes and positioning of components ^[extracted]

## Key Insights

- The core stack is minimal: [[entities/chatgpt|ChatGPT]] images + [[entities/codex|Codex]]. No hand-coding required. ^[extracted]
- This is a concrete example of how [[concepts/vibe-coding|vibe coding]] combines with AI image generation to lower the barrier to producing complete, polished websites. ^[inferred]
- The workflow mirrors the [[concepts/agent-workflow-loop|Agent Workflow Loop]]: tell the agent what to do → assess results → refine with additional prompts. ^[inferred]
- Unlike [[concepts/wireframe-to-app-generation|wireframe-to-app generation]] which starts from hand-drawn sketches, this approach starts from AI-generated photorealistic mockups. ^[inferred]
- The technique generalizes beyond websites — any visual design can potentially be reconstructed by a coding agent when broken down section by section. ^[inferred]
